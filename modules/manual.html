OrbVista Manual

Advanced Volumetric Analysis & Molecular Property Mapping Suite

1. Data Handling & File Management

Import Use the App menu to load various computational formats:

  • Load MO / Dens / ESP Cube: Import standard Gaussian-style cube files.
  • Load Molden / FCHK: Import electronic structure data directly for on-the-fly grid calculation.

File List Interaction: Loaded files appear in the sidebar. Single-click to draw the structure; Double-click to remove a file from the session.

2. Visualization & Analysis

Electronic Properties

  • MOs: Select a Molden/FCHK file to unlock the Alpha/Beta Orbital Tabs.
  • Mapping: Toggle the Scale Bar for ESP or Spin Density mapping.
  • Adjustments: Use the sliders to set Opacity and define v_max / v_min for consistent color scales.

Grid & Calculation

Access Menu -> Grid Settings to define the resolution for new calculations:

  • n_pts: Grid points (nx, ny, nz) for volumetric data.
  • Padding: Buffer zone (in Angström) around the molecule to prevent field cut-offs.

3. Interaction & Customization

Right-Click Plotter Context Menus

CanvasChange background color (Dark/Light mode).
Orbital LobeIndividually change the color of the selected positive/negative lobe.
DensityCustomize the color mapping of the density surface.

4. Generation & Export Options

Draw Menu Calculate Volumetric Data

  • MO / Dens / ESP: Select a Molden or FCHK file to trigger 3D generation.
  • ESP (Cube-based): Select two files (Density + ESP-Potential) to generate a mapped surface from existing data.
  • Spin Mapping: Directly project spin density onto the electron density surface from QC outputs.

Export Menu Save Your Results

  • Images: High-resolution .png screenshots.
  • Rendering: Export scenes as POV-Ray (.inc) or Blender (.glb) files.
  • Analysis: Save .cube files. Note: Exporting an MO/Dens cube also generates a synchronized .xyz file for external tools like ChimeraX.
Scientific Tip: For publication-quality animations, export your reaction segments via POV-Ray. The generated arrays allow for seamless frame-by-frame rendering with consistent isosurfaces.
